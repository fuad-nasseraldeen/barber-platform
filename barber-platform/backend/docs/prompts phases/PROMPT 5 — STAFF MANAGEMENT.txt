Endpoints
Method	Endpoint	Permission	Description
POST	/staff	staff:create	Create staff
GET	/staff	staff:read	List staff (query: businessId, includeInactive)
GET	/staff/:id	staff:read	Get staff by ID
PATCH	/staff/:id	staff:update	Update staff (including deactivate via isActive: false)
DELETE	/staff/:id	staff:delete	Soft delete staff
POST	/staff/services	staff:manage	Assign services to staff
POST	/staff/working-hours	staff:manage	Set working hours
POST	/staff/breaks	staff:manage	Add break
POST	/staff/time-off	staff:manage	Add time off
Guards & RBAC
JwtAuthGuard and RolesGuard on all staff routes
@Roles('owner', 'manager') at controller level
@Permissions() per endpoint (staff:create, staff:read, staff:update, staff:delete, staff:manage)
RolesGuard resolves businessId from params, body, or query, and from staffId when needed
Validation (DTOs)
CreateStaffDto – businessId, firstName, lastName, optional locationId, userId, email, phone, etc.
UpdateStaffDto – businessId, optional fields including isActive for deactivation
DeleteStaffDto – businessId in body
ListStaffQueryDto – businessId (required), includeInactive (optional)
StaffServicesDto, StaffWorkingHoursDto, StaffBreakDto, StaffTimeOffDto – validated per schema
Route order
Specific paths (/staff/services, /staff/working-hours, /staff/breaks, /staff/time-off) are declared before parameterized routes so they are matched correctly.
