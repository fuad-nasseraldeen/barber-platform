4. Design Decisions
Decision	Rationale
Row-level tenancy	Single DB, simpler ops, suitable for thousands of tenants
slotKey unique	Prevents double-booking under concurrency
Staff.userId unique	One User → one Staff (1:1)
Soft delete (deletedAt)	Recoverable deletions on core entities
Composite indexes	(tenantId, startTime), (tenantId, status) for booking queries
