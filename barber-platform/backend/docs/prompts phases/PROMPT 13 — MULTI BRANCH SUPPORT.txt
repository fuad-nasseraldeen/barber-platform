Extend the SaaS platform to support multiple branches per business.

A business may have multiple physical branches.

Update the architecture to support:

Business
└ Branches
└ Staff
└ Appointments
└ Services
└ Waitlist
└ Customers

Requirements:

1. Create a new entity: Branch

Fields:

id
businessId
name
address
city
lat
lng
phone
createdAt

2. Update existing entities to include branchId:

staff
appointments
waitlist
customers
staff_working_hours
staff_breaks
staff_time_off

branchId must be nullable to support existing data.

3. Add API endpoints:

GET /branches
POST /branches
PATCH /branches/:id
DELETE /branches/:id

4. Dashboard updates:

Add branch selector in the dashboard header.

Managers can view:

* a specific branch
* all branches

5. Analytics must support:

branch filter
staff filter
date range

6. Ensure all queries are filtered by businessId and branchId where applicable.

Do not break existing functionality.
